🚀 Guide de démarrage rapide
1
🛡️ Installer WireGuard
WireGuard est le protocole VPN ultra-rapide utilisé par VPN Guard. Il doit être installé sur votre PC. Le lanceur LANCER_VPNGUARD.bat l'installe automatiquement.
2
🔑 Clés serveur (automatique)
VPN Guard génère automatiquement vos clés de chiffrement au premier lancement. Elles sont sauvegardées dans le dossier configs/.
3
👥 Ajouter des Peers (appareils)
Allez dans l'onglet Peers, cliquez sur Ajouter un peer. Chaque peer reçoit une configuration à importer sur l'appareil distant (téléphone, laptop, etc.).
4
⚡ Se connecter
Retournez au Tableau de bord et cliquez sur le grand bouton toggle pour démarrer le tunnel VPN. Vous pouvez aussi utiliser le bouton Connexion rapide.
5
🔒 Activer les protections
Activez le Kill Switch pour bloquer internet si le VPN tombe, et la Protection DNS pour éviter les fuites. Ces options sont dans les onglets dédiés.
📖 Comprendre les fonctionnalités
🔗 Qu'est-ce qu'un VPN WireGuard ? ▶
WireGuard est un protocole VPN moderne, ultra-rapide et sécurisé. Il crée un tunnel chiffré entre votre PC et un autre appareil (ou serveur).

Chiffrement : Curve25519 (clés), ChaCha20-Poly1305 (données), BLAKE2s (hash)
Port : UDP 51820 par défaut
Performance : Beaucoup plus rapide qu'OpenVPN, latence minimale

Tout le trafic internet passe par le tunnel chiffré, ce qui masque votre véritable IP et protège vos données.
🖥️ Mode Serveur vs Mode Client ▶
Mode Serveur (par défaut) :
Votre PC devient le serveur VPN. D'autres appareils (téléphone, laptop) se connectent à vous. Idéal pour accéder à votre réseau domestique depuis l'extérieur.

Mode Client :
Votre PC se connecte à un serveur VPN distant (ex: un VPS, un autre PC). Importez un fichier .conf dans l'onglet Connexion. Tout votre trafic passe par le serveur distant.

Prérequis serveur : Ouvrir le port UDP 51820 sur votre routeur (redirection de port) et connaître votre IP publique.
🔒 Kill Switch — protection anti-fuite ▶
Le Kill Switch ajoute des règles au pare-feu Windows pour bloquer tout le trafic internet si le VPN se déconnecte accidentellement.

Comment ça marche :
1. Bloque toutes les connexions sortantes
2. Autorise uniquement le trafic via le tunnel VPN (interface wg0)
3. Autorise le réseau local (LAN) pour imprimantes, NAS, etc.

Important : Si le Kill Switch reste actif et que le VPN est arrêté, vous n'aurez plus internet. Désactivez-le dans l'onglet Kill Switch.
🌐 Protection DNS — éviter les fuites ▶
Même avec un VPN actif, vos requêtes DNS (traduction nom de domaine → IP) peuvent fuiter hors du tunnel et révéler les sites que vous visitez.

La protection DNS force toutes les requêtes à passer par les serveurs DNS configurés (1.1.1.1, 9.9.9.9) via le tunnel VPN.

Test de fuites : Utilisez le bouton "Vérifier fuites DNS" dans l'onglet DNS Protection pour vérifier que votre IP publique correspond à celle du VPN.
🔀 Split Tunneling — choisir le trafic ▶
Le Split Tunneling permet de choisir quelles applications passent par le VPN et lesquelles utilisent votre connexion directe.

Exemples d'utilisation :
• Navigateur → via VPN (navigation privée)
• Jeux vidéo → bypass (meilleur ping)
• Streaming → bypass (éviter blocage géo inverse)

Note : Cette fonctionnalité est en version beta.
📶 Protection Wi-Fi — réseaux non fiables ▶
VPN Guard peut détecter automatiquement le réseau Wi-Fi auquel vous êtes connecté et activer le VPN si le réseau n'est pas dans votre liste de confiance.

Réseaux de confiance : Ajoutez votre Wi-Fi maison, bureau, etc. Le VPN ne s'activera pas sur ces réseaux.
Réseaux non fiables : Cafés, hôtels, aéroports — le VPN s'active automatiquement pour protéger vos données.
👥 Peers — ajouter des appareils ▶
Un peer est un appareil qui se connecte à votre VPN. Chaque peer a sa propre paire de clés et une IP unique dans le réseau VPN (10.66.66.x).

Comment ajouter un peer :
1. Onglet Peers → Ajouter un peer → Donnez un nom
2. VPN Guard génère automatiquement les clés et la config
3. Cliquez sur 📋 Config pour voir le fichier .conf
4. Importez ce fichier sur l'appareil distant (app WireGuard mobile/desktop)

Apps WireGuard : Disponibles sur iOS, Android, macOS, Linux, Windows.
📁 Profils — gérer plusieurs configs ▶
Les profils permettent de sauvegarder différentes configurations VPN et de basculer rapidement entre elles.

Profils par défaut :
• 🏠 Maison — votre réseau domestique
• 🏢 Bureau — réseau professionnel
• 📶 Wi-Fi Public — configuration sécurisée pour les hotspots

Vous pouvez aussi importer un fichier .conf pour vous connecter à un serveur VPN externe.
🔧 Résolution de problèmes
❌ Je ne peux pas connecter le VPN ▶
Vérifications :
1. WireGuard est-il installé ? → Vérifiez dans le diagnostic ci-dessus
2. Le programme est-il lancé en administrateur ? → WireGuard nécessite des droits admin pour créer le tunnel
3. Utilisez LANCER_VPNGUARD.bat qui gère tout automatiquement

Si rien ne marche :
• Installez WireGuard manuellement : wireguard.com/install
• Ou via la commande : winget install WireGuard.WireGuard
🚫 Le Kill Switch bloque tout internet ▶
Si le Kill Switch est actif et le VPN déconnecté, tout le trafic internet est bloqué. C'est le comportement normal (protection anti-fuite).

Solution :
1. Allez dans l'onglet Kill Switch
2. Désactivez le toggle
3. Les règles pare-feu seront automatiquement supprimées

En cas d'urgence (si l'interface ne répond pas) :
Ouvrez un terminal admin et exécutez :
netsh advfirewall firewall delete rule name=all dir=out program=any
⚠️ Ceci supprime TOUTES les règles sortantes — utilisez en dernier recours.
🌐 Fuite DNS détectée ▶
Une fuite DNS signifie que vos requêtes DNS passent en dehors du tunnel VPN.

Solution :
1. Activez la Protection DNS dans l'onglet dédié
2. Vérifiez que les serveurs DNS sont configurés (1.1.1.1, 9.9.9.9)
3. Relancez le test de fuites
🔌 Pas d'internet après connexion VPN ▶
Causes possibles :
• Mode Serveur : Vérifiez que les règles PostUp/PostDown sont configurées pour le NAT (forwarding)
• Mode Client : Vérifiez que le serveur distant est joignable et que la config .conf est correcte
• DNS : Essayez de changer les serveurs DNS (1.1.1.1 → 8.8.8.8)
• Kill Switch : Désactivez-le temporairement pour tester
🔑 Comment ouvrir le port 51820 sur mon routeur ? ▶
Pour utiliser le mode serveur depuis l'extérieur :

1. Connectez-vous à l'interface de votre routeur (souvent 192.168.1.1)
2. Cherchez Redirection de port (Port Forwarding / NAT)
3. Ajoutez une règle :
• Port externe : 51820
• Port interne : 51820
• Protocole : UDP
• IP interne : L'IP de votre PC
4. Sauvegardez et redémarrez le routeur si nécessaire

Note : Configurez votre endpoint avec votre IP publique dans l'onglet Mode Serveur.
