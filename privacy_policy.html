1. Introduction et responsable du traitement

NetGuard Pro est une suite de cybersécurité locale pour Windows développée par NetGuard (« nous », « notre »). Cette politique décrit comment l'application traite vos données conformément au Règlement général sur la protection des données (RGPD/GDPR, Règlement UE 2016/679).

Responsable du traitement : NetGuard
Email : sxc3030@gmail.com

2. Les adresses IP sont des données personnelles

Conformément à la jurisprudence de la CJUE (affaire C-582/14, Breyer c. Bundesrepublik Deutschland, 19 octobre 2016), les adresses IP sont considérées comme des données personnelles au sens de l'article 4(1) du RGPD, car elles peuvent permettre l'identification indirecte d'une personne physique. NetGuard Pro traite des adresses IP dans le cadre de l'analyse du trafic réseau et les protège en conséquence.

3. Données collectées

3.1 Métadonnées de trafic réseau (traitées localement)

  • Adresses IP — IP source et destination du trafic réseau, analysées pour la détection d'intrusions.
  • Ports et protocoles — numéros de port et protocoles (TCP, UDP, ICMP, etc.) pour la classification des menaces.
  • Journaux de sécurité — alertes, menaces détectées et actions du pare-feu, enregistrées dans des fichiers locaux.
  • Configuration — vos préférences, règles et paramètres stockés dans des fichiers JSON locaux.
  • Identifiants — le mot de passe administrateur est stocké sous forme de hash SHA-256 en local.

Important : Aucun contenu de payload (corps des paquets) n'est stocké de manière permanente. Aucun fichier personnel n'est analysé sans votre consentement explicite.

3.2 Requêtes externes (transferts internationaux)

Pour l'analyse des menaces, NetGuard Pro peut envoyer des adresses IP publiques aux services tiers suivants :

ServiceFinalitéLocalisation
ip-api.comGéolocalisation d'IPAllemagne / États-Unis
VirusTotal (Google)Vérification de réputation d'IPÉtats-Unis
AbuseIPDBFlux de menaces / IP malveillantesÉtats-Unis

Base juridique pour ces transferts : Intérêt légitime (Art. 6(1)(f) RGPD) pour la sécurité réseau, complété par des clauses contractuelles types (CCT) de la Commission européenne lorsque les données sont transférées vers les États-Unis.

3.3 Données NON collectées

  • Aucune télémétrie, aucun analytics, aucun tracking
  • Aucun suivi publicitaire
  • Aucun cookie tiers
  • Aucun envoi de données à nos propres serveurs
  • Aucune analyse de fichiers personnels sans consentement

4. Base juridique du traitement (Art. 6 RGPD)

Le traitement des données réseau par NetGuard Pro est fondé sur l'intérêt légitime (Art. 6(1)(f) RGPD) : la sécurité des réseaux et des systèmes d'information, conformément au considérant 49 du RGPD qui reconnaît explicitement la sécurité réseau comme intérêt légitime.

Nous avons procédé à une mise en balance des intérêts :

  • Intérêt poursuivi : Détection et prévention des intrusions réseau, protection contre les cybermenaces.
  • Nécessité : L'analyse des métadonnées réseau (IP, ports, protocoles) est indispensable au fonctionnement d'un IDS/IPS.
  • Impact sur les droits : Minimal — seules les métadonnées sont traitées, aucun contenu de communication n'est stocké.

5. Conservation des données

Toutes les données sont stockées exclusivement sur votre ordinateur dans le répertoire d'installation :

  • logs/ — journaux de sécurité
  • captures/ — captures réseau
  • reports/ — rapports de sécurité
  • netguard_settings.json — configuration
  • netguard_users.json — comptes utilisateurs (hachés)

Durée de conservation : Les journaux de sécurité et captures sont conservés pendant 30 jours maximum, puis automatiquement purgés. Vous pouvez également les supprimer manuellement à tout moment via le tableau de bord.

6. Sécurité des données

L'application utilise :

  • Hachage SHA-256 pour les mots de passe
  • Tokens de session avec expiration automatique (24h)
  • Communication locale uniquement (localhost)
  • Aucun port ouvert vers l'extérieur par défaut

7. Vos droits (RGPD)

Conformément au RGPD, vous disposez des droits suivants concernant vos données personnelles :

  • Droit d'accès (Art. 15) — Obtenir confirmation que vos données sont traitées et en recevoir une copie.
  • Droit de rectification (Art. 16) — Faire corriger des données inexactes vous concernant.
  • Droit à l'effacement (Art. 17) — Demander la suppression de vos données personnelles. Vous pouvez supprimer tous les journaux et captures via le tableau de bord ou désinstaller l'application.
  • Droit à la limitation du traitement (Art. 18) — Demander la restriction du traitement de vos données dans certaines circonstances.
  • Droit à la portabilité (Art. 20) — Recevoir vos données dans un format structuré, couramment utilisé et lisible par machine (les journaux sont en format texte/JSON).
  • Droit d'opposition (Art. 21) — Vous opposer au traitement fondé sur l'intérêt légitime. Vous pouvez désactiver les requêtes de géolocalisation et de flux de menaces dans les paramètres.

Pour exercer ces droits, contactez-nous à : sxc3030@gmail.com

Droit de réclamation : Vous avez également le droit d'introduire une réclamation auprès d'une autorité de contrôle compétente (en France : CNIL ; au Canada : Commissariat à la protection de la vie privée).

8. Restriction d'âge

NetGuard Pro est un outil de cybersécurité professionnel destiné aux utilisateurs de 16 ans et plus. L'application ne collecte pas sciemment de données auprès de personnes de moins de 16 ans.

2. IP Addresses Are Personal Data

In accordance with the Court of Justice of the European Union ruling (Case C-582/14, Breyer v. Bundesrepublik Deutschland, October 19, 2016), IP addresses constitute personal data under Article 4(1) of the GDPR, as they can enable indirect identification of a natural person. NetGuard Pro processes IP addresses as part of network traffic analysis and protects them accordingly.

3. Data Collected

3.1 Network Traffic Metadata (processed locally)

  • IP addresses — Source and destination IPs from network traffic, analyzed for intrusion detection.
  • Ports and protocols — Port numbers and protocols (TCP, UDP, ICMP, etc.) for threat classification.
  • Security logs — Alerts, detected threats, and firewall actions recorded in local files.
  • Configuration — Your preferences, rules, and settings stored in local JSON files.
  • Credentials — The admin password is stored as a local SHA-256 hash.

Important: No payload content (packet bodies) is permanently stored. No personal files are scanned without your explicit consent.

The processing of network data by NetGuard Pro is based on legitimate interest (Art. 6(1)(f) GDPR): the security of networks and information systems, in accordance with Recital 49 of the GDPR which explicitly recognizes network security as a legitimate interest.

We have conducted a legitimate interest assessment:

  • Interest pursued: Detection and prevention of network intrusions, protection against cyber threats.

  • Impact on rights: Minimal — only metadata is processed, no communication content is stored.

All data is stored exclusively on your computer in the installation directory:

  • logs/ — security logs

  • reports/ — security reports

Retention period: Security logs and captures are retained for a maximum of 30 days, then automatically purged. You can also manually delete them at any time via the dashboard.
